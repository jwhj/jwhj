bzoj4198题解

May 29, 2016 10:02 · 100 words · 1 minute read 题解 bzoj huffman

如果不会huffman树，先去学一学吧。

这道题显然就是一个k叉的huffman树，问题是网上好像很难找得到这样的资料，估计是出题人自己想出来的（？）

如果简单地修改huffman树的构造过程，把每次选两个改成每次选k个，得到的算法是错误的。正确的算法应该是怎么样的呢？

先考虑比较简单的情况，假设最后构造出来的huffman树每个非叶子节点都有k个孩子。这个时候上面的算法可以跑出正确的解。一开始有n棵树，每次选k个出来并成一棵，树的数量减少了(k-1)。最后只剩一棵，所以$k-1|n-1$。

如果最后构造出来的huffman树有些非叶子节点有不到k个孩子，那么这样的节点只应有一个，而且在树的最底部。所以第一次选出节点并成一棵树时要选少一些，准确来说，是(n-1)%(k-1)+1个。

代码：

#include <cstdio>
#include <cstring>
#include <algorithm>
#include <queue>
#define ele long long
using namespace std;
#define maxn 100010
struct node
{
    ele w,h;
    inline bool operator<(node b)const
    {
        if (w<b.w) return false;
        if (w>b.w) return true;
        return h>b.h;
    }
};
ele n,k;
priority_queue<node> q;
int main()
{
    scanf("%lld%lld",&n,&k);
    for (int i=0; i<n; ++i)
    {
        ele w;
        scanf("%lld",&w);
        node p=(node){w,1};
        q.push(p);
    }
    ele ans=0;
    while (q.size()>1)
    {
        ele m=(q.size()-1)%(k-1);
        if (m==0) m=k; else ++m;
        ele w1=0,h1=0;
        for (int i=0; i<m; ++i)
        {
            node p=q.top(); q.pop();
            w1+=p.w;
            h1=max(h1,p.h);
        }
        ans+=w1;
        q.push((node){w1,h1+1});
    }
    printf("%lld\n%lld\n",ans,q.top().h-1);
    return 0;
}
