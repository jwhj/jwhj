bzoj1193题解

Jul 31, 2017 10:02 · 84 words · 1 minute read 题解 bzoj 数学

网上最多见的做法好像都挺玄学的……感觉我的做法比较靠谱而且高效。

一开始的思路跟这道题一样，得到方程： $$(t_1+t_2)+2(t_3+t_4)=x$$ $$(t_3-t_4)+2(t_1-t_2)=y$$

要求$|t_1|+|t_2|+|t_3|+|t_4|$的最小值。

x，y的正负是无关紧要的，所以不妨把x，y都取绝对值。

注意到$|t_1|+|t_2|=\mathrm{max}\{|t_1+t_2|,|t_1-t_2|\}$，即要求$\mathrm{max}\{|x-2p|,|q|\}+\mathrm{max}\{|y-2q|,|p|\}$的最小值，保证x与q同奇偶，y与p同奇偶。（这里假设$t_3+t_4=p,t_1-t_2=q$）

用调整法容易证明，取得最小值的时候x-2p,y-2q,p,q可以都是非负的。讨论一下x-2p与q，y-2q与p的大小关系，像做高中线性规划题那样画一个p，q的图像，可以发现，最小值在2p+q=x,p+2q=y的交点及它们与坐标轴的交点附近取到。

这里是交点附近而不是交点处，是因为p，q只能取整数。其实这个也很好处理，枚举交点处$\pm 3$的所有点就可以了。

代码：

#include <cstdio>
#include <cstring>
#include <algorithm>
#define ele int
using namespace std;
#define K 3
const ele INF=1e9;
ele xp,yp,xs,ys,x,y,ans;
inline ele _abs(ele x){
	return (x<0)?-x:x;
}
inline ele calc(ele p,ele q){
	return max(_abs(x-2*p),_abs(q))+max(_abs(y-2*q),_abs(p));
}
inline void solve(ele p0,ele q0){
	for (int p=p0-K; p<=p0+K; ++p)
		for (int q=q0-K; q<=q0+K; ++q)
			if ((!((x-q)&1)) && (!((y-p)&1)))
				ans=min(ans,calc(p,q));
}
int main(){
	scanf("%d%d%d%d",&xp,&yp,&xs,&ys);
	x=_abs(xp-xs); y=_abs(yp-ys);
	ele p0=(2*x-y)/3,q0=(2*y-x)/3;
	ans=INF;
	solve(p0,q0);
	solve(x/2,0); solve(0,x);
	solve(y,0); solve(0,y/2);
	printf("%d\n",ans);
	return 0;
}
